Subversion

You can retrieve a snapshot of the current development version from the Subversion repository on .

Use the following command to retrieve the current source tree: 

svn co https://relacs.svn.sourceforge.net/svnroot/relacs/relacs/trunk relacs

(this command will download the source files into the directory ./relacs>).

Then change to the relacs directory and read the README and the INSTALL files for more instructions.

You can also browse the Subversion repository.

Supported Platforms

relacs is currently developed under Linux. However, since it is coded in standard C++ and uses the Qt-library for the GUI, a port to Mac OS X and Windows should be possible. For this, it would be helpful to change to Qt version 4 first. Also, interfaces to the (data acquisition) devices need to be implemented for each platform as needed. Any help is appreciated!

Requirements

  • Linux with kernel version 2.6
  • GNU autotools
  • Qt version 3.3 (older versions have a bug in the thread support).
  • Gnu Scientific Library (recommended)
  • doxygen for generating the documentation

Roadmap

Phase 1 - Basic development
We have been continually developing and using relacs in our lab since 2000. Features are added as we need them. This development was and still is driven by the very specific requirements of our experiments (see publications).
Phase 2 - Modularization and generalization
Soon we realized that the structure of the program was way to specialized and we started to better modularize the code and to outsource everything that was specific to our experiments as a plugin. As a result the core program is now completely independent of the specific hardware used and independent of the type of experimental setup.
Phase 3 - Technical issues (version 0.9.6)
Since 2007 we are actively making the program ready to be shared with other people, i.e. setting up GNU autotools and finalizing the interfaces. Since most of this is accomplished by now you are welcome to join the project and contribute to it.
Phase 4 - New urgent features (version 0.9.7)
A couple of new features need to be implemented or finished as soon as possible (most of this by spring-summer 2009):
  • Dynamic clamp support
  • Data browser
  • Plugins for current and voltage clamp recordings
  • Update the plugins for the electric fish experiments
Phase 5 - Usability (version 1.0)
A program that is highly modular and that compiles and runs flawlessly is still hard to use if the documentation is incomplete and configuration files are not explained. Therefore, the following things need to be accomplished before relacs is really usable:
  1. Add a tutorial to the documentation
  2. Add various widgets and wizards for configuring relacs:
    • Macro editor
    • Filter and detector setup tool
    • Data acquisition device and channel selection
